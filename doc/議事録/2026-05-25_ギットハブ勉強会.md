## GitHub / VS Code 運用例共有

実際の運用例として、
VS Code を活用した GitHub 操作について実演を実施。

### 実施内容

- GitHubリポジトリのクローン
- フォルダ作成
- mdファイル作成
- 画像フォルダ管理
- Git操作（add / commit / push）
- ドキュメント管理運用イメージ共有

### 確認できた点

- VS Code上で一通りの操作が完結可能
  参加者の眼の前で、Keiちゃんが、実際に武村TLの手ほどきでやってみたところ、
  VScodeとGitのインストールならびに、GitHubからのクローンにあっさりと成功した。
　さらに、Geminiで作成した、かわいい感じの、おいしいコーヒーの淹れ方、という.mdをpushした
　簡単に出来た！！を、参加者全員で体感した！

- md形式により技術情報の蓄積・共有が容易
　これからの時代の技術者は、.mdファイルをどれだけ作成したか、で価値が決まる
　これまでの世界でも大学の研究者とかは、論文の質と数で、評価されていたと思うが、それと同じ

- GitHub上で履歴管理できるメリットが大きい
　特にAIに文章を作成してもらうことが当たり前となるこれからの時代には、どこが変わったのかわかりにくくなる。
　ますます、GitHubのバージョン管理が必須となる

- 個人管理ではなく、チーム共有資産として活用可能
　これが一番大切。
　個人のスキルを、組織知としてこそ、組織人としてのミッションコンプリート

## 今後の進め方

- 小さくても実際に触ることを優先
　本当に活用出来ることを大多数が実感することができれば、Enterprise版に移行した方が良いと思う

- 各自が試した内容を都度共有
　ハッカソンって知ってるか？
　エンジニアやデザイナー、プランナーなどがチームを組み、特定のテーマに沿って、アイデアや技術的な完成度を競い合い、称え合うイベントのこと。
　今日のように出来たことを披露しあい、活発な質問、意見、称賛が、積極的に飛び交う場。
　そこでは、ただだまって聞いている人には、価値が無いとされる。
　技術部の会議室とは、本来そうあるべきと考える。

- 技術情報をGitHubへ蓄積
- 「分かる人だけが使う」状態にしない
- 日常業務へ徐々に組み込んでいく

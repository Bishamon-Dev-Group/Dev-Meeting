# AGENTS.md

## Codex 作業方針

- 基本の回答・説明は日本語で行う。
- 作業前に `README.md` や既存ファイル構成を確認し、このリポジトリの流儀に合わせる。
- 変更は依頼内容に必要な範囲へ絞り、無関係な整形・リファクタリング・ファイル移動は行わない。
- 既存の未コミット変更は利用者の作業として扱い、明示指示なしに戻さない。
- 破壊的な Git 操作（`reset --hard`、強制 checkout、強制 push など）は、明示的な依頼がない限り行わない。
- 実装後は可能な範囲で構文チェック、テスト、または動作確認を行い、結果を簡潔に報告する。
- 外部公開される可能性を前提に、秘密情報・個人情報・認証情報をコミットしない。

## ファイル編集ルール

- 既存の命名、ディレクトリ構成、コードスタイルを優先する。
- コメントは必要最小限にし、コードから分かる内容を繰り返さない。
- Markdown は読みやすさを優先し、リンク切れや相対パスのズレに注意する。
- 日本語ファイル名や日本語パスを扱う場合は、リンク先の実ファイル名と表記を必ず一致させる。

## 確認ポイント

- 変更したファイル一覧
- 実行した確認コマンドと結果
- 未確認事項や、利用者側で必要な commit / push などの次アクション

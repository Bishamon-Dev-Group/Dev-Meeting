# 技術部 開発ミーティング

## 概要

技術部 開発ミーティングの議事録・検討資料を管理するリポジトリです。


## 目的

- 要素技術開発・商品開発テーマの課題解決と意思決定を行うミーティングとする
- 各テーマの進捗を共有し、スケジュール管理へと繋げる
- 議論の経緯と決定事項を開発グループ全体へ共有し、組織の財産として記録する


## 議事録の書き方

1. minutes/ フォルダに YYYY-MM-DD_テーマ名.md のファイル名で作成する
2. templates/meeting-minutes.md をコピーして記入する
3. コミットメッセージは「docs: YYYY-MM-DD ○○ミーティング議事録追加」の形式とする

## 運用ルール

- 議事録はミーティング翌営業日までにプッシュする
- 決定事項に変更があった場合は該当ファイルを更新し、変更履歴をコミットメッセージに残す
- 機密性の高い内容（取引先の具体的な見積金額、未公開の特許情報等）は記載しない
- 画像・図面を添付する場合はリンク形式とする

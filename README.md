# 🛠️✨ 技術部 開発ミーティング ✨🛠️

> **Welcome!** ここは技術部の開発ミーティングに関する議事録や検討資料を、みんなできれいに管理するリポジトリです。一歩ずつ、素敵なプロダクトを作っていこうねっ！(🎀•͈ᴗ•͈)

---

## 🌸 ミーティングの目的（Goal）

メンバー全員のハッピーな開発と、スムーズな意思決定のために3つの目的を大切にしています。

* 💡 **課題解決＆意思決定**：要素技術や商品開発テーマのピンチをチャンスに変える場にします！
* 📈 **進捗ハッピー共有**：みんなの「今ここ！」をシェアして、無理のないスケジュールをキープします。
* 📝 **未来へのたからもの**：議論のプロセスや大事な決定事項をしっかり記録して、グループ全体の財産にします。

---

## ✍️ 議事録の書き方（How to Write）

ルールを守って、だれが見ても「かわいい＆わかりやすい」議事録を作っちゃおう！

1. 📂 `minutes/` フォルダの中にファイルを作ります。
2. 🏷️ ファイル名は `YYYY-MM-DD_テーマ名.md` にしてね。
3. 📄 `templates/meeting-minutes.md` をコピペして可愛く埋めていきましょう！
4. 🚀 コミットメッセージは以下のきまり。
   * `docs: YYYY-MM-DD ○○ミーティング議事録追加`

---

## 🎀 運用ルール（Rules）

みんなが気持ちよく開発をすすめるための、たいせつなお約束です。

* ⏰ **スピード重視**：ミーティングの**翌営業日まで**に、忘れずプッシュしてね！
* 🔄 **アップデート**：決定事項が変わったらファイルを更新して、メッセージに変更内容をメモ！
* 🔒 **ひみつの約束**：取引先のお見積もりや未公開の特許など、大切な機密情報は書かないでね。
* 🖼️ **お写真・図面**：画像を載せたいときは、すっきり綺麗に見せるために**リンク形式**で貼ってね。
